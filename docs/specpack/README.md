# Candor spec-pack — v1 (hand-distilled)

**What this is.** A machine-consumable distillation of the Candor language for a
model that must *generate correct `.cnr` code* — the P19 "specification pack"
(grammar, semantics summary, idiom catalog, diagnostic taxonomy). It is terse and
deduplicated on purpose: a model's context budget is the design constraint (P13
applied to documentation). Load the whole pack; it is sized to fit.

**Files.**
- `grammar.md` — the real surface syntax by example (items, types, modes,
  expressions + precedence, patterns, contextual keywords).
- `semantics.md` — the rules a generator must not violate, ranked by observed
  agent friction (move/borrow/drop/effects/contracts/generics/`?`/statics).
- `idioms.md` — canonical patterns from the corpus, one exemplar each.
- `diagnostics.md` — the full error-code taxonomy (code → meaning → fix); also
  the P4 diagnostic-taxonomy artifact.

**P19 role.** This is the seed deliverable of the model-competence pipeline
(§8 sequencing: seed now, full pipeline against Bet 1 evidence). Its immediate
customer is the project's own agents writing `.cnr`. The synthetic-corpus
generator and eval harness are later stages; this pack is their spec anchor.

**Sources of truth (freshness contract).** Regenerate this pack whenever the
language changes. Authority order (LANG_PHILOSOPHY.md §9): philosophy > design
docs > spec chapters > implementation. Grammar/lexis: `docs/spec/01`,`02` +
design `0006`. Semantics: `docs/spec/03`–`08` + designs `0001`,`0004`,`0005`.
Generics/iteration: designs `0007`,`0009`. Corpus: `compiler/tests/fixtures/**`.
Diagnostics: mined from `compiler/src/`. When spec and implementation disagree,
`grammar.md`/`semantics.md` say which is ahead (see the "Divergences" section in
each).

**Versioning honesty.** This is **v1, hand-distilled** — written by a model
reading the sources, not generated by the P19 pipeline. It reflects the prototype
edition (no FFI, one valve, `enforced`-only contracts, generics stages 1–3). It
can drift; the pipeline will produce validated later versions.
