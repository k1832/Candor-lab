# Adversarial review #1 of design 0013 (text-type budget) — (b) accept after revision

**Date:** 2026-07-09. Two-directional attack (too-large / too-small). **Verdict: (b).** The budget
is correctly sized both ways: not too large (str+String+[u8]+contract is minimal; the CStr/OsStr/
Cow/char-scalar/encoding-genericity refusals all pass the recorded-justification test), not too
small (the deferrals are principled). The requested high-value check — char iteration "fits
neither 0009 protocol" — VERIFIES correct, with a sharper reason: str-implements-Iter needs no
borrow field, so the SOLE blocker is 0009's uniform alloc marker on Iter::next, and the sole fix
is a dedicated non-alloc Chars protocol; deferral justified per 0009's own pressure-case
methodology.

Dispositions (all accepted; applied same-commit):
1. **MAJOR (soundness) — push(u32) could forge an ill-formed str.** push had no validity contract;
   push(s, 0xD800)/0x110000 encodes to bad bytes → false str via as_str, breaking the invariant at
   the primary builder. Fixed: push carries `enforced requires(is_scalar_value(c))` — the char-as-
   u32 cost paid at the single u32→text door, statically dischargeable when decoded from a P7
   boundary.
2. **SOUNDNESS-of-argument — CStr "buys nothing" overclaimed.** The no-interior-NUL half is
   checkable on Candor-owned bytes (Rust's CStr::new enforces it). Fixed: §1.4 keeps the TYPE
   refusal but routes no-interior-NUL to `enforced requires(no_interior_nul(s))` on the Candor→C
   wrapper, not assumed-proven; the "buys nothing" claim softened to the honest not-zero cost of
   keeping the checkable half enforced by discipline.
7. **MINOR — char_count** clarified to ship WITH the deferred Chars protocol, not the minimal
   surface.
8. **MINOR — char-iteration blocker** re-attributed to the uniform alloc marker alone (OBL-ITER-
   BORROW was a red herring for the Iter case).

Held: two-named-types budget and its sizing (both directions), the CStr and char-scalar type
refusals, the substr P5 fault choice, the mechanical literal migration (a "..."→str retype is a
hard type error at annotated [u8] sites, never silent drift — verified against the parser
fixture), the freestanding/non-UTF-8 story (non-text bytes stay [u8], losing no str op), and the
str-not-a-field / portability claims.
